The Problem

Manual catalog cloning is fragile and error-prone.

⚙
Manual SQL Scripts
No automation — hand-written CREATE TABLE AS SELECT for every object.
🔒
Missing Metadata
Permissions, tags, and properties silently dropped on every clone.
❌
No Validation
Row counts unchecked — data drift goes unnoticed for weeks.
📄
No Audit Trail
Compliance gaps — no record of what was cloned, when, or by whom.

Feature Overview

Nine pillars of enterprise-grade catalog management.

Clone Wizard

Get Started

Three paths to running Clone-Xs in your environment.

From Source
Full stack with UI, API, and CLI.
git clone https://github.com/​your-org/Clone-Xs.git
cd Clone-Xs
pip install -e ".[dev]"
make web-start
Docker
One command — API + UI ready.
docker-compose up --build
CLI Only
Lightweight — just the clone engine.
pip install clone-xs

clone-catalog clone \
  --source edp_dev \
  --dest edp_dev_00
