The Problem

Cloning Unity Catalog catalogs today is manual, fragile, and incomplete

Manual SQL Scripts

Teams write custom SQL to clone tables one-by-one. No automation, no error handling, no rollback.

Missing Metadata

Permissions, tags, properties, ownership, constraints, and comments are silently dropped.

No Validation

No way to verify clone accuracy — row counts, checksums, and schema parity are unchecked.

No Audit Trail

Nobody knows who cloned what, when, or whether it succeeded. Compliance gaps everywhere.

Result

Cloning a single catalog takes hours, fails silently, and leaves teams without confidence in their data.

Serverless Compute

Run clones on Databricks serverless — no SQL warehouse needed
How It Works

1. Package & Upload

Clone-Xs builds a Python wheel and uploads it to your selected UC Volume.

2. Generate Notebook

Creates a runner notebook that imports the wheel and executes the clone with all options.

3. Submit Job

Submits a one-time Databricks job using serverless compute. No cluster provisioning.

4. Stream Results

Polls job status and streams output back to the UI in real-time.

Benefits
$0
No Warehouse Costs
Auto
Scales Automatically
Fast
No Cluster Start Wait

CLI Usage

clone-catalog clone
  --source edp_dev
  --dest edp_dev_00
  --serverless
  --volume /Volumes/cat/schema/vol

Incremental Sync

Sync only changed tables using Delta version history — no full re-clone
How It Works
  • Compares Delta table versions between source and destination
  • Identifies tables with new commits since last sync
  • Syncs only changed tables — skips up-to-date ones
  • Supports schema-level and table-level selection
  • Parallel processing across schemas
  • Runs on serverless or SQL warehouse

Selection Modes

  • All schemas — scan entire catalog
  • Selected schemas — pick specific schemas
  • Selected tables — per-table checkboxes
  • Select all / Clear — bulk operations

Getting Started

Up and running in 3 commands
Option 1: From Source
git clone github.com/viral0216/clone-xs
cd clone-xs
pip install -e ".[dev]"
make web-start
Frontend on :3000 • API on :8000
Option 2: Docker
docker-compose up --build

# Open http://localhost:8000
Full stack in one container
Option 3: CLI Only
pip install -e .
clone-catalog clone \
  --source my_catalog \
  --dest my_catalog_clone
No server needed for CLI usage

Required Setup After Install

  • Connect to Databricks — Enter workspace URL + PAT in Settings
  • Select SQL Warehouse — Choose from dropdown (or use serverless)
  • Initialize Audit Tables — Optional: set audit catalog for Delta logging
